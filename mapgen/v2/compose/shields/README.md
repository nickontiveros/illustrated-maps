# Highway shield references

Highway shields are rendered as **AI sprites** generated by the asset studio in
the map's painted style, just like trees, buildings, and ornaments. They are not
bundled final art.

To make a generated shield reproduce the *correct* placard shape and colors, the
asset studio is given a **reference image**: a blank (number-less) shield. Those
reference images live in `refs/` and are fetched from Wikimedia Commons.

## How references are sourced

`mapgen/v2/compose/shields.py` maps an OSM `network` tag to a Commons "blank
shield" file and fetches a server-side-rendered PNG of it (no local SVG
rasterizer needed):

| Network        | Commons file                |
| -------------- | --------------------------- |
| `US:I`         | `File:I-blank.svg`          |
| `US:US`        | `File:US blank.svg`         |
| `US:<STATE>`   | `File:<State name> blank.svg` (e.g. `Arizona blank.svg`) |

The fetch happens during the **plan** stage (`fetch_shield_reference`), which is
already online for OSM, and the PNG is cached here as `refs/<network-slug>.png`
(e.g. `refs/us-az.png`) and committed. Later runs — and the plan builder used in
tests — resolve references **cache-only** (`cached_shield_reference`), so they
never touch the network.

A network with no reference (unknown state, non-US network, or an offline first
run) simply gets a **procedurally drawn** shield instead — the reference is
purely additive.

## The number is always code-drawn

References are *blank* shields. The route number is drawn on top by the
compositor in a clean bold font, so the image model never renders text.
